ILMSpace (yoki CourseSpace)
Название проекта: [Рабочее название, например, "IlmSpace"]
Тип: Маркетплейс образовательного контента (LMS). Платформа: Web (Адаптивный сайт: Desktop + Mobile).
1. Основные роли пользователей
    1. Студент: Потребляет контент.
    2. Автор (Учитель): Создает контент.
    3. Администратор (Ты): Управляет всем, модерирует курсы, видит финансы.
2. Функционал MVP (Что делаем обязательно)
А. Модуль Регистрации и Авторизации:
    • Регистрация по номеру телефона (SMS через Eskiz или PlayMobile) — критично для Узбекистана!
    • Вход через Google/Telegram (желательно).
    • Восстановление пароля.
Б. Витрина (Главная страница):
    • Поиск по ключевым словам.
    • Категории (IT, Языки, Хобби, Ремесло).
    • Карточка курса: Название, Автор, Цена, Рейтинг, Кнопка «Купить».
В. Страница Курса (Лендинг курса):
    • Видеотрейлер (бесплатный просмотр).
    • Описание: «Чему вы научитесь», «Требования».
    • Программа курса (список уроков).
    • Об авторе.
Г. Кабинет Студента (LMS Player):
    • Плеер для просмотра видео (с защитой от простого скачивания).
    • Список уроков с пометками «Просмотрено».
    • Текстовые материалы под видео.
    • Домашние задания (просто текстовое поле для отправки ответа).
Д. Кабинет Автора:
    • Конструктор курса: Добавить модуль -> Добавить урок -> Загрузить видео.
    • Установка цены.
    • Дашборд: «Мои продажи», «Баланс».
Е. Платежный модуль:
    • Интеграция Click, Payme, Uzum Pay (API).
    • Биллинг: История транзакций.
Ж. Админ-панель (Back-office):
    • Модерация: Одобрить/Отклонить новый курс перед публикацией.
    • Управление пользователями (бан/разбан).
    • Финансы: Видеть, сколько должны авторам, сколько заработала платформа.
3. Технические требования (Stack)
Программист спросит, на чем писать. Вот рекомендации для стартапа (быстро и масштабируемо):
    • Backend (Сервер): Python (Django/FastAPI) или Node.js. Это стандарты рынка.
    • Frontend (Лицо сайта): React.js или Vue.js. (Обязательно SPA — чтобы сайт работал быстро, как приложение).
    • База данных: PostgreSQL.
    • Хранение видео: Важный момент! Не хранить видео просто на сервере (забьется память). Использовать стриминговые сервисы (например, Vimeo PRO, BunnyCDN или AWS S3) для защиты от пиратства.
